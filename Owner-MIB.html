Owner-MIB-2 module-identity fdOwnerMIB

This MIB defines object types related to the definition of owners for entries in tables with an iniial owner index. Administrators can limit access to table entires based on the owner value. Owner-MIB-4 object-type fdOwnerTable

1 Owner table

This table is used to allow an administrator to manage the owners that can create entries within tables where fdOwnerIndex is listed as the first index. Owners should be granted read-access to their specific rows so that they can discover their administratively defined limits and to discover points of discontinuity within their counter objects. Some tables need to support the definition of rows specific to a manager. For example, it might not be appropriate to allow one manager to change the configuration of a event logging mechanism that was defined by another manager. Such tables should be structured with an initial index of fdOwnerIndex. While several tables might need this capability, the list of allowed owners can be managed by a single administrator through the use of this MIB. All rows of this table shall be stored in non-volatile memory.

 not-accessible

Owner-MIB-7 object-type fdOwnerIndex

2 Owner index

This object provides a unique one-octet, non-zero, integer identifer for the owner, which can be used as the first index into a table. Identifiers do not have to be assigned sequentially.

 not-accessible

Owner-MIB-10 object-type fdOwnerRowStatus

5 Owner row status

This object indicates the status of this row within the fdOwnerTable. Objects within this row can be modified regardless of the state of this object. The destruction of a row within this table shall also result in the destruction of all associated rows within tables that use the fdOwnerIndex as an index. The creation and deletion of rows within this table also results in the automatic creation and deletion of rows in any supported tables that AUGMENT this table.

Making an owner inactive shall result in all rows in tables having the associated fdOwnerIndex value as an index inactive as well.
